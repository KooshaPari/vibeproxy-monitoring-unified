# Contributing to vibeproxy-monitoring-unified

Thanks for your interest. This project is part of the [Phenotype](https://github.com/KooshaPari) ecosystem.

## Workflow

1. Fork the repository.
2. Create a topic branch off `main`: `git checkout -b feat/short-description`.
3. Make focused commits (Conventional Commits preferred: `feat:`, `fix:`, `docs:`, `chore:`).
4. Run local quality checks (lint, format, test) before pushing.
5. Open a Pull Request against `main` with a clear description.

## Code of Conduct

By participating, you agree to abide by the [Code of Conduct](./CODE_OF_CONDUCT.md).

## Reporting Issues

Use [GitHub Issues](https://github.com/KooshaPari/vibeproxy-monitoring-unified/issues)
with a minimal reproduction and your environment details.

## Security

For security-sensitive disclosures, see [SECURITY.md](./SECURITY.md) instead of filing a public issue.
